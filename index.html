Movigo.js

Ultralight JS library to animate your DOM elements. For the installation of the library read the README file on Github. If you want to try the library go to Ceditor.

  • ✔ Light (~3kb)
  • ✔ Modern
  • ✔ Easy to use
‣

Functions

First function to call is always target, which takes a selector or a DOM element as input and return an object of functions. We can divide these functions into action functions and option functions (or simply actions and options). Every action or option return other actions and options, with a final function: animate, that plays the animation and return a promise (fulfilled at the and of animation). Click on `play` button at top left to play animations. 


    const target = movigo.target('div')

    // or:

    const element = document.querySelector('div')
    const target = movigo.target(element)

    target.duration(.8) // Option function.

    target.to({ width: '200px' }) // Action function.

    // Create the animation and wait for the end.
    await target.duration(.8).to({ width: '200px' }).animate()
‣

Method chaining

As you can see Movigo use method chaining technique, and save the state of the animation in each function. This means that you can break the chain and create new chains with additional functions by building different animations. 


    const animation = target.duration(.4)

    await animation.to({ width: '100px' }).animate()
    await animation.to({ height: '100px' }).animate()
‣

Actions

Actions allow you to change CSS property values. There are currently two function: to and from. Both take an object containing CSS properties and they work like in a CSS keyframe: 'from' represent the initial state of the animation, 'to' the final state. If 'from' function is not defined, the initial state is represented by the current CSS target properties. To see a full list of available actions you can also use movigo.actions() function. 


    await target.duration(.3).to({
      width: '80px',
      height: '80px',
      borderRadius: '100%',
      transform: 'rotateZ(90deg) translateY(-100px)'
    }).animate()
‣

Options

Options allow you to set some properties of the animations, such as duration, speed curve of the transition effect, delays or loops. In particular, a loop can be infinite or determined by a number. To see a full list of available options you can use movigo.options() function. 


    await target
        .delay(.5).duration(.8)
        .easing('ease-in-out')
        .loop(2)
        .to({
            width: '80px',
            height: '80px',
            transform: 'rotateZ(180deg)'
        }).animate()

    // Infinite loop:
    // target.loop().to({ left: '100px' }).animate()

Specific options

Option functions can get a function as parameter, in which i-th element can be used to define element specific options. 


    const letters = element.querySelectorAll('span')
    const target = movigo.target(letters)

    await target
        .delay((i, el) => i * .05).duration(.4)
        .easing('cubic-bezier(0.4, 0.0, 0.2, 1)')
        .to({ opacity: 1 }).animate()

Feedback

You can create simple animations to make your UI more user-friendly with feedback. 


    await target
        .duration(.1).loop(1)
        .easing('cubic-bezier(0.33, 1, 0.68, 1)')
        .to({ transform: 'scale(2, 2)', opacity: .3 })
        .animate()
‣

Loading

You can create loading animations with a few lines of code. 


    await target
        .delay(i => i * .1).duration(.4).loop(5)
        .to({ transform: 'scaleY(1.5)', opacity: .6 })
        .animate()
‣

Lists

Setting duration, easing and delays correctly you can create interesting animations for the lists. 


    const elements = element.querySelectorAll('li')
    const target = movigo.target(elements)

    await target
        .delay(i => i * .1).duration(.3)
        .easing('cubic-bezier(0.0, 0.0, 0.2, 1)')
        .to({ opacity: 1, transform: 'translateY(0px)' })
        .animate()
‣

Focus

By running multiple animations simultaneously it is possible to create more complex effects, such as a greater focus on an element (e.g. images). 


    selectedElement.style.zIndex = '2'

    movigo.target(selectedElement)
        .duration(.3)
        .easing('cubic-bezier(0.34, 1.56, 0.64, 1)')
        .to({ transform: 'scale(2.2, 2.2)' })
        .animate()

    for (const element of otherElements) {
        element.style.zIndex = '1'
    }

    movigo.target(otherElements)
        .duration(.3).easing('ease-in-out')
        .to({ filter: 'brightness(60%) blur(1px)' })
        .animate()
‣

Choreography

Simultaneous animations and correct parameters also allow you to create complex choreographies, in which the elements move in harmony. 


    movigo.target(screen)
        .duration(.3)
        .to({ transform: 'scale(1, 1)' })
        .animate()

    movigo.target(element.nextElementSibling.querySelector('img'))
        .duration(duration)
        .from({ borderRadius: '100%', filter: 'blur(3px)' })
        .to({ borderRadius: '0', filter: 'blur(0)' })
        .animate()

    movigo.target(list)
        .duration(.3)
        .from({ opacity: 0 })
        .to({ opacity: 1 })
        .animate()
